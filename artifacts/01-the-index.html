Rules

  1. Refreshed on every ingest — counts, listings, stubs, and contradictions all update together.
  2. Each line is one wiki-link plus a short hook. No narrative, no paragraphs.
  3. Incidents are newest-first; services, root-causes, concepts, owners are alphabetical.
  4. The trailing two sections surface debt — stubs that need writing, contradictions that need resolving.
  5. If a section is empty, keep the heading and write (none). Don't delete sections.
  6. A stale Last refreshed: header means the wiki has drifted. Run an ingest or a lint.

Sections — in order

  1. Summary — one paragraph: what happened, who was affected, how it ended.
  2. Timeline — bullet list, HH:MM — event, in UTC.
  3. Detection — how it was noticed (alert, customer report, self-reported).
  4. Root cause — link to a root-causes/ page; brief description specific to this incident.
  5. Resolution — what mitigated, what fixed.
  6. Action items — - [ ] owner — task — due. Completed items stay, struck through.
  7. Notes — open questions, contradictions, links to related incidents.

Rules

  1. File name is {YYYY-MM-DD}-{kebab-slug}.md. Date = start of incident in UTC.
  2. Type-specific frontmatter: date, severity, duration_minutes, services, root_causes.
  3. Link every named service, root-cause, and owner — first mention is a wiki-link.
  4. If a referenced entity has no page, create a stub rather than leave a dangling link.
  5. Unknown owner? Write owner: unknown and flag it under Notes.

## Notes
Shares pager rotation with [[orders-team]] on weekends.

Sections — in order

  1. Members — roles and rotation only. No names, emails, or handles.
  2. Services owned — bullet list, every service a wiki-link.
  3. Recent incidents — newest first, wiki-linked.
  4. Notes — cross-team rotations, on-call quirks, anything else.

Rules

  1. File name is the team slug in kebab-case, e.g. payments-team.md.
  2. Type-specific frontmatter: services, incidents.
  3. Never include personal contact info. Names, emails, phone numbers, chat handles — none of it. Point to the directory if needed.
  4. Recent incidents list mirrors what the incident pages claim in their owner: field. If they drift, lint will flag.
  5. Owner unknown? Don't create an empty owner page — leave owner: unknown on the consuming page and flag it.
